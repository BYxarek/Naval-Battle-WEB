# План разработки морского боя

## Цель

Сделать красивую браузерную игру "Морской бой" для ПК на React с мультиплеером между разными компьютерами, анимациями, комнатами по коду и сервер-авторитетной игровой логикой на PHP 8.2 + MySQL 8.0 без отдельного Node-сервера.

## Этапы

1. Подготовить базовую архитектуру.
   - `Vite 8.0.8` + `React 19.2.5` + `TypeScript 6.0.2`
   - backend на `PHP 8.2` + `MySQL 8.0`
   - синхронизация матча через polling API
   - общая игровая модель в `shared/`

2. Зафиксировать правила классического режима.
   - поле `10x10`
   - корабли `4, 3, 3, 2, 2, 2, 1, 1, 1, 1`
   - по одному выстрелу за ход
   - корабли не пересекаются и не касаются друг друга даже по диагонали
   - повторный выстрел в ту же клетку запрещён
   - победа при уничтожении всех 5 кораблей соперника

3. Реализовать сетевую модель.
   - создание комнаты
   - вход по коду
   - состояние двух игроков
   - готовность после расстановки
   - серверная валидация выстрелов и смены хода
   - восстановление комнаты по локальному токену игрока

4. Реализовать UX постановки кораблей.
   - выбор корабля
   - смена ориентации
   - ручная постановка
   - случайная расстановка
   - визуальный hover-preview

5. Реализовать матч.
   - два поля: своё и поле соперника
   - индикация попаданий, промахов и потопленных кораблей
   - история событий
   - состояние хода
   - экран победы/поражения

6. Сделать выразительный визуальный стиль.
   - насыщенный морской фон
   - цветовые слои, свечение, блики
   - заметные, но быстрые анимации
   - адаптация под широкие и узкие экраны

7. Проверить проект.
   - установка зависимостей
   - `npm run build`
   - `php -l api/*.php`
   - ручной прогон локальной сессии после загрузки схемы БД

## Ближайшие улучшения

- список активных комнат
- чат в комнате
- звуковые эффекты
- таймер хода
- приватные приглашения и переподключение
- режим реванша
