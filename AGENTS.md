# AGENTS.md

## Контекст проекта

Проект: мультиплеерный "Морской бой" для ПК.

Технологии:
- `React 19.2.5`
- `Vite 8.0.8`
- `TypeScript 6.0.2`
- `PHP 8.2`
- `MySQL 8.0`
- `motion 12.38.0`
- `zustand 5.0.12`

Архитектура:
- клиент в `src/`
- API в `api/`
- SQL-схема в `database/`
- общая игровая логика и типы в `shared/`

## Цели продукта

- дать двум игрокам на разных ПК играть через комнату по коду
- придерживаться классических правил морского боя
- сделать интерфейс ярким, современным и анимированным
- не переносить критичную игровую логику на клиент
- не требовать отдельный Node-процесс на хостинге

## Правила игры

- поле `10x10`
- флот каждого игрока: `1x4`, `2x3`, `3x2`, `4x1`
- корабли ставятся горизонтально или вертикально
- выход за границы, пересечения и касание по стороне или диагонали запрещены
- выстрел строго один за ход
- повторно стрелять в ту же клетку нельзя
- при попадании раскрывается только факт попадания; тип корабля раскрывается при потоплении
- победа наступает при уничтожении всего флота соперника

## Правила для ИИ-агентов

- сначала читать `shared/game.ts`, затем `api/index.php`, затем клиентские компоненты
- не ломать сервер-авторитетную модель
- не переносить проверку правил только в UI
- не раскрывать координаты чужих кораблей в API-ответах
- сохранять единый стиль: насыщенная палитра, морская тема, заметная типографика, мягкие glow-эффекты
- не добавлять новые библиотеки без явной причины
- если меняются правила игры, обновлять и `PLAN.md`, и этот файл
- при изменении API синхронно обновлять PHP-обработчики и клиентский `fetch`-слой
- при изменении постановки кораблей сохранять возможность ручной и случайной расстановки
- после любых изменений сразу выполнять `npm run release` и оставлять актуальный архив в `release/`

## Кодовые соглашения

- использовать TypeScript `strict`
- общие типы держать в `shared/`
- UI-компоненты делать небольшими и специализированными
- избегать расхождения правил между `shared/game.ts` и `api/game.php`
- предпочитать чистые функции для игровых расчётов

## Что проверять после изменений

- создаётся ли комната
- может ли второй игрок подключиться по коду
- проходит ли постановка кораблей без нелегальных раскладок
- стартует ли игра только после готовности обоих игроков
- правильно ли меняется ход
- не раскрываются ли чужие корабли раньше времени
- завершается ли матч корректной победой
