# Морской Бой

Текущая версия проекта: `1.0.0`

Браузерная игра в морской бой с PHP/MySQL-бэкендом, матчами между разными устройствами, режимом против серверного бота и релизом под обычный shared hosting без постоянного Node-процесса.

## О проекте

Проект сделан как полноценная онлайн-игра:

- два игрока подключаются к одной комнате по коду
- можно начать отдельную игру против серверного бота
- каждый расставляет свой флот по правилам
- ходы синхронизируются через PHP API и polling
- матч работает на обычном CGI/PHP-хостинге без Node.js-сервера
- есть светлая и тёмная темы, настройки языка и графики боя
- интерфейс адаптирован под разные размеры экранов и устройства

## Основные возможности

- создание комнаты и вход по коду
- отдельная игра с ботом без реального второго игрока
- ручная расстановка флота с preview и проверкой правил
- авторасстановка флота с ручным подтверждением
- старт боя только после готовности всех игроков
- запрет касания кораблей даже по диагонали
- повторный ход после попадания или потопления
- перезапуск таймера после каждого нового хода, включая попадание и потопление
- серверный бот, который добивает найденный корабль после попадания
- уведомления об ошибках и игровых событиях
- сдача во время боя
- отмена комнаты на этапе расстановки
- экран финальной статистики
- реванш только по согласию двух игроков
- мгновенный новый матч против бота через тот же серверный контракт комнаты
- мобильное боковое меню в матче
- локализация интерфейса и серверных сообщений: `ru`, `en`, `uk`
- presence heartbeat и счётчик онлайна
- PWA-кнопка для добавления сайта на главный экран телефона

## Правила флота

Поле: `10x10`

Состав флота:

- `1` корабль на `4` клетки
- `2` корабля на `3` клетки
- `3` корабля на `2` клетки
- `4` корабля на `1` клетку

Корабли не могут соприкасаться друг с другом ни сторонами, ни углами.

Во время боя:

- нельзя стрелять в одну и ту же клетку дважды
- при промахе ход переходит сопернику
- при попадании или потоплении игрок ходит снова
- тип корабля раскрывается только при потоплении
- победа наступает после уничтожения всего флота соперника

## Локальный запуск

Установить зависимости:

```bash
npm install
```

Запустить фронтенд в режиме разработки:

```bash
npm run dev
```

Для локальной игры также нужен доступный PHP API и настроенная база данных. Параметры подключения обычно хранятся в `api/config.local.php`, этот файл не коммитится.

Собрать production-версию:

```bash
npm run build
```

Подготовить релиз для хостинга:

```bash
npm run release
```

## Релиз

Команда `npm run release`:

- собирает фронтенд
- готовит папку `release/morskoy-boy/`
- копирует туда нужные файлы для хостинга
- создаёт архив `release/morskoy-boy-hosting.zip`

Этот архив можно сразу загружать на хостинг.

## Структура проекта

```text
src/                  React-клиент
src/api/              Fetch-слой, room API, запросы и session
src/components/       Экраны и композиция UI
src/components/game/  Боевые и setup-компоненты
src/components/lobby/ Создание комнаты, вход по коду, bot-room
src/hooks/roomSync/   Polling и синхронизация состояния комнаты
src/styles/           Foundation, theme и responsive CSS
api/                  PHP API и модульная серверная логика
api/actions/          Create/join/setup/battle/rematch actions
shared/               Общие типы и контракты комнаты
shared/game/          Чистые правила боя, board helpers и авторасстановка
database/             SQL-схема базы данных
scripts/              Release, shared, contract, cleanup, smoke и UI-скрипты
public/               Manifest, robots и favicon
release/              Готовые архивы и staging релиза
```
